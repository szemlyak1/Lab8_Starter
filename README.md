How are graceful degradation and service workers related?

Graceful degradation is the practice of building an application so that it continues to provide a baseline level of functionality even when modern features or critical resources fail. Service workers are a perfect example of this concept in action. Service workers are just graceful degradation actually happening. Service workers ensure that if a user loses their internet connection—or experiences a very slow one—the application doesn't completely break down or display a generic browser error. Instead, the app "degrades gracefully" by functioning offline using the locally stored cache.
